모듈 통합 감정 분석

CIRCLE은 텍스트 전처리부터 복합 감정 분석까지 11개의 전문 모듈이 유기적으로 연결된 고도화된 감정 분석 시스템입니다.

11개 감정 분석 모듈

hub

1. ComplexEmotionAnalyzer

복합 감정 통합 오케스트레이터

세그멘테이션, 그래프 분석, 충돌 해결, 랭킹, 클러스터링, 임베딩 생성, 품질 검증, 전이 패턴 분석을 통합한 핵심 파이프라인입니다.

  • 문서 세그멘테이션 및 전처리
  • 감정 그래프 구성 및 분석
  • 감정 충돌 해결 및 우선순위 랭킹
  • 클러스터링 및 임베딩 벡터 생성
  • 품질 검증 및 전이 패턴 분석
핵심 기능 : 전체 파이프라인 오케스트레이션
신뢰도 임계값: 0.10 (MAD 기반 자동 조정)
처리 성능: 500MB 메모리 제한, 캐시 최적화
explore

2. ContextExtractor

맥락/상황 추출 + 흐름 신호

텍스트의 문맥과 상황을 추출하여 감정 분석의 정확도를 높이고, 시간적/공간적/사회적 맥락을 분석합니다.

  • 시간적 맥락 추출 (언제)
  • 공간적 맥락 추출 (어디서)
  • 사회적 맥락 추출 (누구와)
  • 트리거 요소 식별 (왜)
  • 진행적 맥락 분석
정규식 패턴 : 시간, 장소, 사회, 트리거 매칭
매칭 정확도: 0.6 이상 유사도 임계값
컨텍스트 윈도우: 3문장 범위 분석
account_tree

3. EmotionRelationshipAnalyzer

감정 간의 관계와 상호작용을 분석하여 복합적인 감정 상태를 파악하고 소셜 감정 그래프를 구성합니다.

  • 감정 간 상관관계 분석
  • 감정 전이 패턴 탐지
  • 감정 호환성 측정
  • 소셜 감정 그래프 구축
  • 감정 쌍 식별 및 강도 계산

4. EmotionIntensityAnalyzer

각 감정의 강도를 정밀하게 측정하고 시간적/상황적 강도 변화를 분석하여 감정의 깊이와 정도를 수치화합니다.

  • 감정 강도 스케일링 (0-100)
  • 상대적 강도 비교
  • 시간적 강도 패턴 분석
  • 상황적 강도 변화 추적
  • 강도 임베딩 생성
측정 범위: 0-100 스케일 + 임베딩
강도 임계값: Low: 0.2, Medium: 0.5, High: 0.7
상대 강도: 최대값 대비 비율 계산

강도 분석 알고리즘

강도 계산 공식
I(e) = W(e) × S(e) × C(e)

I(e): 감정 e의 최종 강도 (0-100)

W(e): 가중치 (키워드 빈도 × TF-IDF 점수)

S(e): 신호 강도 (패턴 매칭 신뢰도)

C(e): 컨텍스트 보정 계수 (0.8-1.2)

강도 레벨 분류 임계값
LOW 0-33점 0 ≤ I(e) < 0.33
MEDIUM 34-66점 0.33 ≤ I(e) < 0.67
HIGH 67-100점 0.67 ≤ I(e) ≤ 1.0

고객 이탈을 사전에 차단하는 작동 원리

1
실시간 텍스트 수집

상담사와 고객의 대화를 실시간으로 수집합니다. 문장 단위로 KSS(Korean Sentence Splitter) 알고리즘을 적용하여 자연스러운 문장 경계를 정확히 파악합니다.

예시: "상품이 늦게 도착했어요" / "정말 화가 나네요" → 두 문장으로 분리
→
2
감정 분류 및 강도 측정

각 문장을 120가지 세부 감정으로 분류하고, 강도를 0-100점으로 측정합니다. TF-IDF 가중치와 임베딩 유사도를 결합하여 정확도를 높입니다.

분석 결과: 분노 60% + 실망감 25% + 불만 15% = 강도 87점 (HIGH)
→
3
전이 패턴 탐지

이전 문장과 현재 문장의 감정 변화를 추적합니다. "긍정 → 부정" 급변을 감지하면 즉시 알림을 발송합니다.

감지 패턴: 락(만족) → 노(분노) = 급변 → 이탈 위험 85%
→
4
위험도 산출 및 검증

부정 감정 비율, 감정 강도, 전이 패턴을 종합하여 이탈 위험도를 0-100%로 계산합니다. 엔트로피 검증으로 신뢰도를 확인합니다.

산출식: (노+애 비율 × 강도) + (전이 급변 보너스) × 신뢰도 = 최종 위험도
→
5
맞춤형 대응 전략 제시

감정 종류, 강도, 전이 패턴을 분석하여 상황별 최적 대응 전략을 자동 생성합니다. 상담사 스킬 레벨과 도메인 특성을 고려합니다.

권장 조치: "즉시 고위급 상담사 전환" + "보상안 제시 (배송비 환불 + 쿠폰)"

핵심 차별점: 단순 키워드 매칭이 아닌, 감정 전이 패턴과 컨텍스트 분석을 통해 고객의 진짜 심리를 파악합니다. 이로 인해 "환불하고 싶어요"라는 표현에서도 고객이 결정적 단계에 이르렀는지, 아니면 여전히 설득 가능한 단계인지를 정확히 판단할 수 있습니다.

  • EmotionAI는 텍스트 속 감정을 정확하게 분석하는 고도화된 AI 시스템입니다. 11개의 전문 모듈(ComplexEmotionAnalyzer, ContextExtractor, EmotionRelationshipAnalyzer, EmotionIntensityAnalyzer, LinguisticMatcher, PatternExtractor, PsychologicalCognitiveAnalyzer, SituationAnalyzer, EmotionSequenceAnalyzer, TransitionAnalyzer, EmotionWeightCalculator)이 협력하여 복합 감정, 강도, 문맥 등을 종합적으로 분석합니다. 120가지 세부 감정을 희(기쁨), 노(분노), 애(슬픔), 락(즐거움) 4차원으로 분류하며, TF-IDF 스코어링과 CUDA 가속을 통해 실시간 분석이 가능합니다.

  • EmotionAI는 감정 분류, 강도 측정(0-100 스케일), 복합 감정 탐지, 문맥 분석, 전이 패턴 분석, 심리학적 해석, 시간 시퀀스 분석 등 다양한 감정 분석 작업을 수행합니다. 특히 감정 변화 과정을 문장 단위로 추적하며 안정/진동/급변 등의 흐름 패턴을 자동 분류합니다. 전이 임계값은 기본 0.4이며 MAD 기반 자동 조정이 가능하고, 신뢰도 임계값은 0.10부터 시작하여 각 감정의 신뢰도를 정밀하게 측정합니다.

  • 한국어 텍스트를 중심으로 다양한 형태의 텍스트를 분석할 수 있습니다. 대화, 리뷰, 소셜미디어 게시물, 고객 피드백 등 감정이 담긴 모든 텍스트를 처리합니다. 특히 컨택센터 상담 대화, 금융권 고객 상담, 기업 CS 상담 등 전문 분야에서 높은 정확도를 보입니다. KSS를 사용한 한국어 문장 분할과 3문장 윈도우 컨텍스트 분석을 통해 더욱 정확한 감정 분석이 가능합니다.

  • 정확도를 높이기 위해서는 다음과 같은 환경 설정을 권장합니다:
    1. 파이프라인 프리셋: $env:EA_PIPELINE_PRESET = "heavy"로 설정하여 11개 모듈 전체를 활성화합니다.
    2. 감정 알ias 모드: $env:EMOTION_ALIAS_MODE = "strict_neutral_lak"로 설정하여 엄격한 분석을 수행합니다.
    3. 온도 보정: $env:ENABLE_TEMP_SCALING = "1"로 활성화하여 확률 보정을 적용합니다.
    4. 커버리지 설정: $env:QUICK_COVERAGE = "strict"로 설정하여 엄격한 커버리지로 품질을 향상시킵니다.
    5. 적응형 임계값: $env:ADAPTIVE_THRESHOLD = "1"로 활성화하여 동적 임계값 조정을 수행합니다.

  • 임베딩 생성을 위해서는 python -m src.generate_embeddings --mode exact --preset demo --oneclick --from-talk 명령어를 사용합니다. 환경 변수로 $env:EMB_JSONL_FLUSH_EVERY = "10"을 설정하면 실시간으로 진행 상황을 확인할 수 있습니다.
    라벨 생성을 위해서는 python -m src.build_labels_programmatic --oneclick --preset simple --session . --allow-no-session --emotions "src/EMOTIONS.json" --out "data/labels.full.jsonl" 명령어를 사용합니다. $env:LABEL_FLUSH_EVERY = "50"으로 설정하면 N개마다 fsync하여 안정성을 확보합니다. 자동 재개 기능이 있어 중단된 작업을 같은 명령어로 이어서 실행할 수 있습니다.
